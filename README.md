# ZIG SIM — iOS

Real-time sensor streaming app for iPhone/iPad. Streams data from device sensors over network protocols (OSC/JSON) to remote systems such as DAWs, visual software, and game engines.

Developed by [1→10, Inc.](https://1-10.com)

---

## Features

| Category | Sensors / Features |
|---|---|
| Motion | Acceleration, Gravity, Gyroscope, Quaternion |
| Position | GPS, Compass, Altitude (Barometer) |
| Touch | Multi-touch, Apple Pencil |
| AR / Vision | ARKit (device / face / image / body tracking), Image Detection (face, QR, rectangle, text) |
| Device | Battery, Proximity, Audio Level (microphone) |
| Connectivity | NFC Reader, iBeacon |
| Input | Remote Control (media keys) |
| Streaming | NDI (Network Device Interface) |

Data is transmitted as OSC bundles or JSON over TCP or UDP.

## Setup

### Repository structure

This repository is public and available to fork. However, `Private/` is a git submodule linked to a private repository that is not accessible to the general public. It contains the source code for paid features: NDI, ARKit, ImageDetection, and NFC.

Forking and building this public repository without the `Private/` submodule is supported. Features that depend on `Private/` will not be available, but the rest of the app will build and run normally.

### Clone

```bash
git clone --recursive https://github.com/1-10/ZIGSIMPlus_iOS_Public.git
```

### Open

Open `ZIGSIMPlus.xcworkspace` in Xcode (not `.xcodeproj`).

### Build

Select a real device target and build. Most sensor features do not work in the simulator.

## Architecture

Three-tab MVP/MVVM architecture:

```
Tab 1: Select     → CommandSelectionViewController / Presenter
Tab 2: Monitor    → CommandOutputViewController / Presenter
Tab 3: Settings   → CommandSettingViewController / Presenter
```

**Data flow:**

```
User selects Commands
  → ServiceManager activates sensor services
  → Each Service collects data
  → Data formatted as OSC messages or JSON
  → NetworkAdapter sends via TCP / UDP
  → CommandOutputViewController shows live preview
```

**Key modules:**

| Module | Role |
|---|---|
| `Command.swift` | Enum of 18 sensor/feature types |
| `ServiceManager` | Aggregates active sensor services |
| `CommandPlayer` | Orchestrates streaming loop |
| `NetworkAdapter` | TCP/UDP transport (Network.framework) |
| `AppSettingModel` | Persisted app configuration |
| `Services/` | One service class per sensor |

**Code generation:** `AppSettingModelAutoSave.generated.swift` is generated from `AppSettingModelAutoSave.stencil` via Sourcery.

## Network Protocol

**OSC message path format:**

```
/ZIGSIM/{deviceUUID}/{sensorAddress}
```

**Device metadata** is included in each bundle: device UUID, device name, iOS version, screen dimensions.

**Settings** (configurable in-app):
- Target IP address and port
- Transport: TCP or UDP
- Format: OSC or JSON
- Sampling rate: 1 / 10 / 30 / 60 Hz

## Development

### Branch strategy

| Branch | Purpose |
|---|---|
| `master` | Main branch — direct commits prohibited |
| `legacy` | Snapshot of the pre-modernization codebase |
| `future/issue-<n>-<desc>` | Short-lived feature branches |

- Always branch from `master`
- Never commit directly to `master`
- 1 PR = 1 Issue; open as Draft first

### Linting and formatting

```bash
# SwiftLint
swiftlint

# SwiftFormat
swiftformat .
```

### Tests

```bash
xcodebuild test -workspace ZIGSIMPlus.xcworkspace \
  -scheme ZIGSIMPlus \
  -destination 'platform=iOS Simulator,name=iPhone 16'
```

Note: sensor-dependent tests require a real device.

### Areas requiring human review

The following changes must be reviewed by a human before merging:

- Code signing / provisioning / certificates
- Entitlements and permissions (camera, location, Bluetooth, NFC)
- Push notification configuration
- Major storyboard or xib changes
- Objective-C bridging
- Secrets and API keys
